### E1: Incremental view-on-view cascades

`propagate_changes` today: table delta applied incrementally to direct
dependents, then every view-on-view does reset() + full re-apply of all
source state — O(full state) per sync, the one place the O(Δ) claim is
false (found in the pre-E borrow audit).

Fix: single topological pass with a pending-deltas map keyed by source
name. Each view applies the pending delta of every updated source; its own
resulting delta joins the map for its dependents. Multi-source updates
accumulate into an arena copy (a second apply_changes clears the view's
delta, so downstream needs the union); single-source updates borrow
`get_delta()` directly (topological order guarantees no re-apply before
the read — established by the pre-E lifetime audit). The reset path is
deleted, including its table re-reads (drift self-healing is not a
feature; drift would be a bug).

Correctness: multi-level cascade differential (join → aggregate → sort
over two mutating tables, delete-heavy) + existing cascading tests + soak
(its aggregate view already cascades from the join view). Perf: bench case
timing single-row syncs through a 3-level chain over a 50k-row base —
must drop from O(n) to O(Δ).

### E2: Correlated subqueries (DELIM_JOIN)

Probe first (optimizer-off DELIM_JOIN + DELIM_GET shapes for correlated
EXISTS / scalar). Design after probing; expected: DELIM_JOIN = join of the
outer relation with a per-outer-key aggregate/lookup of the inner subplan
where DELIM_GET scans the distinct correlated keys. Likely circuit shape:
distinct-keys view feeding the inner subplan (keyed), joined back. If the
faithful translation exceeds a session, land the most common patterns
(correlated EXISTS / IN / scalar agg comparison via GROUP-BY-rewrite
equivalence) and keep E110 for the rest — decided by what the probes show.

### E3: DuckDBRow hash caching

Remaining 2.4× eval gap = Z-set inserts rehashing every column Value.
Cache the hash in DuckDBRow (mutable lazily-computed field, invalidated
never — rows are immutable once inside Z-sets; construction sites that
mutate columns must not have cached yet, enforce by computing lazily in
DuckDBRowHash). Bench before/after; suite + soak green.
